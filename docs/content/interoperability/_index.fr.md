Dans cette section, l'accent est mis sur ce qui peut être fait pour réduire la dépendance aux fournisseurs
et changer facilement de fournisseur de LLM.

## Utiliser des standards

Utilisez des standards pour réduire la dépendance aux fournisseurs et garantir des changements
plus faciles vers des concurrents ou des alternatives auto-hébergées sans perdre de temps sur la reconfiguration de l'équipe.

- [**Agent Client Protocol**](https://agentclientprotocol.com/get-started/introduction) : n'importe quel LLM sur n'importe quel éditeur (ou extension), changez de fournisseur de LLM sans modifier votre configuration.
- [**Agent Skills**](https://agentskills.io/home) : construisez des *skills* standardisés utilisables par tous les agents.
- [**AGENTS.md**](https://agents.md) : un nom de fichier d'instructions simple pour tous les agents (ou presque).
- [**Model Context Protocol**](https://modelcontextprotocol.io/docs/getting-started/intro) : connectez n'importe quel agent à n'importe quel système externe compatible.
- [**Open Plugin**](https://open-plugins.com) : partagez des composants dans un seul format utilisable par tous les agents.

## Privilégier l'Open Source

Privilégiez les projets, composants, agents et outils *Open Source* sans dépendance à un fournisseur de LLM.

- [**agentgateway**](https://agentgateway.dev) : proxy open source pour le trafic MCP, Agent-to-Agent, LLM et API, fournissant sécurité, observabilité et gouvernance clé en main.
- [**Goose**](https://goose-docs.ai) : agent IA open source et extensible (CLI, application de bureau, API) qui fonctionne avec n'importe quel fournisseur de LLM.

Les deux projets sont hébergés au sein de l'[**Agentic AI Foundation**](https://aaif.io) (AAIF), une fondation neutre nouvellement créée
(décembre 2025), soutenue par la Linux Foundation, dédiée aux standards et projets d'IA agentique ouverts.

Ce qui démarre ici ressemble beaucoup à la [**Cloud Native Computing Foundation**](https://www.cncf.io/) (CNCF).
